
Демо к вебинару **«17. Делегаты, DSL и вспомогательные конструкции Kotlin»**.

Каждый пример — самодостаточный класс со своим `main()`. Запуск: зелёная иконка в IDE рядом с `main` или `./gradlew :webinar-17:run` для конкретного класса.

| №  | Имя класса               | Тема вебинара            | Что демонстрирует                                    |
|----|--------------------------|--------------------------|------------------------------------------------------|
| 1  | `LazyStatsDemo`          | Стандартные делегаты     | `by lazy` — ленивое вычисление с кэшированием         |
| 2  | `LoggingRepository`      | Делегирование класса     | `by delegate` — композиция вместо наследования        |
| 3  | `UserProfile`            | Стандартные делегаты     | `Delegates.observable` — реакция на изменение         |
| 4  | `GameScore`              | Стандартные делегаты     | `Delegates.vetoable` — валидация перед присваиванием  |
| 5  | `Config`                 | Делегированные свойства  | Свойства из `Map` по имени-ключу                      |
| 6  | `NonBlank`               | Делегированные свойства  | Свой делегат: `getValue` / `setValue`                 |
| 7  | `HtmlBuilder`            | Инструменты DSL          | Лямбда с получателем — основа DSL                     |
| 8  | `MenuBuilder`            | Построение DSL           | `@DslMarker` — контроль областей видимости            |
| 9  | `ServerConfig`           | Инструменты DSL          | `apply` — компактный конфигурационный DSL             |
| 10 | `SmartcastContractDemo`  | Контракты                | `returns … implies` — контракт и smartcast            |
| 11 | `BankAccount`            | require / check / assert | Три проверки условий и их назначение                  |
| 12 | `AnnotationDemo`         | Аннотации                | `@Suppress` — метаданные для компилятора/IDE          |
